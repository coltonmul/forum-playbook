07
USAGE RULES
01
Logo positions only
⎇ FORUM PLAYBOOK is a visual mark. Use it in headers, footers, and nav. Never in running text, file names, or anything a search engine reads. SEO-facing text is always "Forum Playbook."
02
ADA always
All text on linen must pass 4.5:1 contrast minimum. Never use #A89880 or lighter as body text. Ghost button text is always #4A4038. Dust (#C4B8A8) is decorative only.
03
Content stays in Drive
Resources live in Google Drive. Videos live on YouTube. The site auto-syncs. Never add content by editing code. File names drive sort order: prefix "00_" to pin to top.
04
Two-wave hover only
Every interactive element uses the locked wave system. Warm family lands on Teal. Cool family lands on Blue. No other hover styles. No exceptions for "simpler" buttons.
05
No frameworks
The site is vanilla HTML, CSS, and JS. No React, Vue, or build step. No Webflow, Squarespace, or CMS. This is intentional. Keeps costs at ~$1.25/month forever.
06
Doc versioning
Every resource document carries a version number (v1.0, v2.1), category label, and version in the header. Update the version when content changes. Format: vMAJOR.MINOR.
07
Document body copy
Body copy in all resource documents is set in Roboto (400 for body, 700 for bold). Header and footer chrome is Consolas. Never use DM Sans or DM Mono in printed or downloaded documents.
